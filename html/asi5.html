Classes and Objects

A class is a user defined< data-type used to bind and associated functions together. It allows the data(and functions) to be hidden, from external use. When defining a class, we are creating a new Abstract Data Type(ADT)that can be treated like any other built- in data type.

The class body contains the declaration of variables and functions. These functions and variables are collectively called as members of the class. They are usually grouped under two visibility lables, namely, private and public.

The members that have been declarred as private can be acessed only by the member functionns. such data hiding using private declaration is the key feacture of object- oriented programming. The use of the keyword private is optional. By default, member of a class are privaate. Public members can be accessed from outside the class also.

An object is an instance of a class. It can be uniquely identified by its name. It has physical existence in the memory. Once a class has been declared, we can creae its objects in the same way as varibles of any other type. Thus, an object is the varible of type class. Eaxmple:

a class student is declared then we can create object of name as, Student s;

Characteristics of a Constructor function:

  1. Its name is same as name of its parent class.
  2. It gets invoked implicitly when an object of its class is created.
  3. It should be declarred in pubilc section of the class.
  4. It does not have return-type,(not even void)
  5. It can have default arguments
  6. It can be overloaded.
  7. It can not be Inherited.
  8. It can not be a virtual function.
  9. It can not contain argument of the same class to which it belongs.

Characteristics of a Friend Function

  1. Unlike member function, a friend function is not invoked using an object name and the dot operator. It is invoked like a normal function.
  2. It can not aceess memberr names directly. It needs to use dot operator and object name to access members of that object. (e.g. s.rollno)
  3. It can not use this keyword.
  4. It may be declared in any ( private or public) part of the class.
  5. A friend function has at least one argument which is of the type of the class(of which it is a friend).
  6. A class can be friend of many classes.
  7. Member funtion of one class may be friend function of another class.
  8. Even a class can be friend of another class. In such case, all member functions of the friend class can access private members of the class.
  9. It is not in the score of the class to which it has been declared as friend.
  10. A friend function is not inherited.

Access Modifiers

There are three types of access modifiers in C++.

  • Public: A member declared with public access modifier is accessible outside the class.
  • Private: A member declared with private access modifier is not accessible outside the class except its friend functins.
  • Procted: A member declared with protected access modifier is not accessible outside the class its friend functions and derived classes.
